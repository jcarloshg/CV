# Estrategias de Agregación y Pivot en SQL

## Agregación

### Funciones de Agregación

Las funciones de agregación realizan cálculos sobre un conjunto de valores y devuelven un solo resultado.

| Función   | Descripción               | Ejemplo       |
| --------- | ------------------------- | ------------- |
| `COUNT()` | Cuenta el número de filas | `COUNT(*)`    |
| `SUM()`   | Suma todos los valores    | `SUM(salary)` |
| `AVG()`   | Calcula el promedio       | `AVG(salary)` |
| `MAX()`   | Valor máximo              | `MAX(salary)` |
| `MIN()`   | Valor mínimo              | `MIN(salary)` |

### GROUP BY

Agrupar filas por uno o más valores:

```sql
SELECT
    department_id,
    COUNT(*) as employee_count,
    AVG(salary) as avg_salary,
    SUM(salary) as total_salary
FROM employees
GROUP BY department_id;

-- Output:
-- department_id | employee_count | avg_salary | total_salary
-- --------------+-----------------+------------+-------------
-- 10            | 5               | 6000       | 30000
-- 20            | 3               | 7500       | 22500
-- 30            | 4               | 5500       | 22000
```

### HAVING

Filtrar grupos después de la agregación:

```sql
SELECT
    department_id,
    AVG(salary) as avg_salary
FROM employees
GROUP BY department_id
HAVING AVG(salary) > 50000;

-- Output:
-- department_id | avg_salary
-- --------------+------------
-- 20            | 7500
-- 40            | 6200

-- Note: Departments 10 and 30 (avg 6000 and 5500) are filtered out
```

### Agregación con CASE (Filtrado Condicional)

```sql
SELECT
    department_id,
    SUM(CASE WHEN status = 'active' THEN 1 ELSE 0 END) as active_count,
    SUM(CASE WHEN status = 'inactive' THEN 1 ELSE 0 END) as inactive_count
FROM employees
GROUP BY department_id;

-- Output:
-- department_id | active_count | inactive_count
-- ---------------+--------------+----------------
-- 10             | 3            | 2
-- 20             | 5            | 0
-- 30             | 2            | 3
```

### Multiple Aggregations

```sql
SELECT
    department_id,
    COUNT(*) as total,
    MAX(salary) as max_salary,
    MIN(salary) as min_salary,
    AVG(salary) as avg_salary,
    SUM(salary) as sum_salary
FROM employees
GROUP BY department_id;
```

## PIVOT

### ¿Qué es PIVOT?

PIVOT transforma valores de filas en columnas, creando una tabla pivote (crosstab).

### PIVOT en SQL Server

```sql
SELECT * FROM (
    SELECT
        department_id,
        year,
        salary
    FROM employees
) AS src
PIVOT (
    AVG(salary)
    FOR year IN ([2022], [2023], [2024])
) AS pvt;

-- Output:
-- department_id | 2022    | 2023    | 2024
-- --------------+---------+---------+--------
-- 10            | 5000    | 5500    | 6000
-- 20            | 7000    | 7500    | 8000
```

### UNPIVOT

Invierte PIVOT: convierte columnas en filas.

```sql
SELECT
    department_id,
    year,
    salary
FROM (
    SELECT
        department_id,
        [2022] as y2022,
        [2023] as y2023,
        [2024] as y2024
    FROM (
        SELECT department_id, year, salary
        FROM employees
    ) AS src
    PIVOT (
        AVG(salary)
        FOR year IN ([2022], [2023], [2024])
    ) AS pvt
) AS unpvt_src
UNPIVOT (
    salary FOR year IN (y2022, y2023, y2024)
) AS unpvt;

-- Output (converts pivoted columns back to rows):
-- department_id | year | salary
-- ---------------+------+--------
-- 10            | 2022 | 5000
-- 10            | 2023 | 5500
-- 10            | 2024 | 6000
-- 20            | 2022 | 7000
-- 20            | 2023 | 7500
-- 20            | 2024 | 8000
```

## Estrategias de Agregación

### 1. Agregación Directa

```sql
SELECT
    category,
    SUM(amount) as total
FROM sales
GROUP BY category;
```

### 2. Agregación con Filtrado

```sql
SELECT
    category,
    SUM(CASE WHEN region = 'North' THEN amount ELSE 0 END) as north_total,
    SUM(CASE WHEN region = 'South' THEN amount ELSE 0 END) as south_total
FROM sales
GROUP BY category;
```

### 3. Agregación con Window Functions

```sql
SELECT
    category,
    amount,
    SUM(amount) OVER (PARTITION BY category) as category_total,
    SUM(amount) OVER () as grand_total,
    SUM(amount) OVER (PARTITION BY category ORDER BY date) as running_total
FROM sales;
```

### 4. Agregación con ROLLUP

Genera subtotales jerárquicos:

```sql
SELECT
    COALESCE(category, 'Total') as category,
    COALESCE(region, 'Subtotal') as region,
    SUM(amount) as total
FROM sales
GROUP BY ROLLUP (category, region);

-- Output:
-- category | region   | total
-- ---------+----------+-------
-- A        | North    | 100
-- A        | South    | 150
-- A        | Subtotal | 250
-- B        | North    | 200
-- B        | Subtotal | 200
-- Total    |          | 450
```

### 5. Agregación con CUBE

Genera todas las combinaciones posibles de subtotales:

```sql
SELECT
    COALESCE(category, 'Total') as category,
    COALESCE(region, 'Total') as region,
    SUM(amount) as total
FROM sales
GROUP BY CUBE (category, region);
```

### 6. Agregación con GROUPING SETS

Especifica múltiples niveles de agregación:

```sql
SELECT
    category,
    region,
    SUM(amount) as total
FROM sales
GROUP BY
    GROUPING SETS (
        (category, region),
        (category),
        (region),
        ()
    );
```

## Estrategias de Pivot

### 1. PIVOT Manual con CASE

```sql
SELECT
    department_id,
    MAX(CASE WHEN year = 2022 THEN salary END) as salary_2022,
    MAX(CASE WHEN year = 2023 THEN salary END) as salary_2023,
    MAX(CASE WHEN year = 2024 THEN salary END) as salary_2024
FROM employees
GROUP BY department_id;

-- Output:
-- department_id | salary_2022 | salary_2023 | salary_2024
-- --------------+-------------+-------------+------------
-- 10            | 5000        | 5500        | 6000
-- 20            | 7000        | 7500        | 8000
```

### 2. PIVOT Dinámico (Procedimiento Almacenado)

```sql
DECLARE @columns NVARCHAR(MAX), @sql NVARCHAR(MAX);

SELECT @columns = STRING_AGG(QUOTENAME(year), ', ')
FROM (SELECT DISTINCT year FROM employees) AS years;

SET @sql = N'
SELECT * FROM (
    SELECT department_id, year, salary
    FROM employees
) AS src
PIVOT (
    AVG(salary)
    FOR year IN (' + @columns + ')
) AS pvt;';

EXEC sp_executesql @sql;
```

### 3. Pivot con Múltiples Valores

```sql
SELECT
    department_id,
    MAX(CASE WHEN year = 2023 THEN salary END) as max_salary_2023,
    MIN(CASE WHEN year = 2023 THEN salary END) as min_salary_2023,
    COUNT(CASE WHEN year = 2023 THEN 1 END) as count_2023
FROM employees
GROUP BY department_id;
```

## Técnicas Avanzadas

### Running Total (Acumulado)

```sql
SELECT
    date,
    amount,
    SUM(amount) OVER (ORDER BY date) as running_total
FROM transactions;

-- Output:
-- date       | amount | running_total
-- -----------+--------+--------------
-- 2024-01-01 | 100    | 100
-- 2024-01-02 | 150    | 250
-- 2024-01-03 | 200    | 450
```

### Moving Average (Promedio Móvil)

```sql
SELECT
    date,
    amount,
    AVG(amount) OVER (
        ORDER BY date
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) as moving_avg_3
FROM transactions;
```

### Rank dentro de Grupos

```sql
SELECT
    department_id,
    employee_id,
    salary,
    RANK() OVER (PARTITION BY department_id ORDER BY salary DESC) as dept_rank,
    DENSE_RANK() OVER (ORDER BY salary DESC) as overall_rank
FROM employees;
```

### Porcentaje del Total

```sql
SELECT
    category,
    amount,
    CAST(amount AS FLOAT) / SUM(amount) OVER () * 100 as pct_of_total
FROM sales;
```

## Ejercicios Prácticos

### Ejercicio 1: Resumen de Ventas por Trimestre

```sql
SELECT
    product_name,
    SUM(CASE WHEN quarter = 1 THEN amount END) as Q1,
    SUM(CASE WHEN quarter = 2 THEN amount END) as Q2,
    SUM(CASE WHEN quarter = 3 THEN amount END) as Q3,
    SUM(CASE WHEN quarter = 4 THEN amount END) as Q4,
    SUM(amount) as yearly_total
FROM sales
GROUP BY product_name;
```

### Ejercicio 2: Comparativa Año Anterior

```sql
SELECT
    year,
    sales,
    LAG(sales, 1) OVER (ORDER BY year) as prev_year,
    sales - LAG(sales, 1) OVER (ORDER BY year) as growth
FROM annual_sales;
```

### Ejercicio 3: Top N por Grupo

```sql
SELECT * FROM (
    SELECT
        department_id,
        employee_id,
        salary,
        ROW_NUMBER() OVER (PARTITION BY department_id ORDER BY salary DESC) as rn
    FROM employees
) WHERE rn <= 3;
```

## Resumen de Estrategias

| Estrategia         | Caso de Uso                     |
| ------------------ | ------------------------------- |
| `GROUP BY`         | Agregación básica por categoría |
| `HAVING`           | Filtrar grupos agregados        |
| `CASE + SUM`       | Pivote manual                   |
| `PIVOT`            | Pivot automático (SQL Server)   |
| `ROLLUP`           | Subtotales jerárquicos          |
| `CUBE`             | Todas las combinaciones         |
| `GROUPING SETS`    | Múltiples niveles de agregación |
| `Window Functions` | Agregación sin perder filas     |
| `Running Total`    | Acumulados                      |
| `Moving Average`   | Tendencias suavizadas           |

---

## Diferencias Clave

- **GROUP BY vs Window Functions**: GROUP BY reduce filas; Window mantiene el detalle
- **PIVOT vs CASE**: PIVOT es más limpio; CASE es más flexible
- **ROLLUP vs CUBE**: ROLLUP es jerárquico; CUBE es completo
- **RANK vs DENSE_RANK**: RANK tiene huecos; DENSE_RANK no
