## Guardrails AI

**Guardrails AI** es una de las herramientas más calientes en el stack de **AI Engineering** actual. Es un framework de código abierto diseñado para añadir una capa de **seguridad, validación y control** a las respuestas de los LLMs (como GPT-4, Llama 3 o Claude) antes de que lleguen al usuario final.

En términos de **MLOps**, es el "firewall" de tu modelo generativo.

### 1. El Concepto: ¿Por qué lo necesitas?

Los LLMs son probabilísticos y a veces impredecibles. Pueden alucinar, revelar datos sensibles (PII) o responder con un tono inadecuado. **Guardrails AI** actúa como un filtro que:

1.  **Valida:** Revisa si el output cumple con reglas específicas (ej. "que sea un JSON válido", "que no tenga insultos").
2.  **Corrige:** Si el output falla, puede intentar corregirlo automáticamente o pedirle al LLM que lo re-genere.
3.  **Garantiza Estructura:** Asegura que el modelo siempre devuelva el formato que tu código espera (muy útil para integrar LLMs con APIs de Python).

### 2. ¿Cómo funciona técnicamente? (RAIL)

Utiliza un formato llamado **RAIL** (Reliability Answer Iteration Logic), que es un archivo XML donde defines:

- **Output:** El esquema que esperas (ej. una lista de strings).
- **Validators:** Las reglas de validación (ej. `is-unbiased`, `valid-sql`, `no-pii`).
- **Prompt:** La instrucción para el modelo.

> **Ejemplo:** Si pides un JSON y el LLM devuelve texto plano, Guardrails lo detecta, lo frena y le dice al LLM: _"Te pedí un JSON, corrígelo"_.

### 3. Pasos para implementarlo en tu Pipeline

Si te preguntan en la entrevista cómo asegurarías la calidad en tiempo real:

- **Definir la "Guardia":** Identificar qué riesgos quieres mitigar (alucinaciones, toxicidad, fugas de datos).
- **Integración en el Código:** Envolver la llamada al LLM con la librería de Guardrails.
- **Acciones de Fallo:** Configurar qué pasa si falla (¿lanzar error?, ¿re-intentar?, ¿devolver un mensaje predefinido?).
- **Logging:** Registrar cada vez que un "guardrail" se activa para analizarlo en tu dashboard de observabilidad (Data Drift semántico).

### 4. Tips & Tricks para la entrevista

- **"No es solo Filtro":** Menciona que no solo filtra palabras feas, sino que valida la **fidelidad semántica**. Por ejemplo, puede verificar si una respuesta está soportada por un documento (similar a lo que hace RAGAS pero en tiempo real).
- **Latencia:** Sé honesto y menciona que añadir Guardrails puede añadir unos milisegundos de latencia, pero es el precio a pagar por la **seguridad y confiabilidad** (Reliability).
- **Pydantic Integration:** Menciona que Guardrails se integra de maravilla con **Pydantic**, lo cual es música para los oídos de un Tech Lead que use FastAPI.

### Cómo decirlo mañana:

> "Para garantizar la operación continua y confiable que pide la vacante, implementaría **Guardrails AI**. Esto nos permite definir restricciones programáticas sobre las respuestas de los modelos, asegurando que siempre cumplan con el formato esperado y pasen filtros de seguridad como la detección de **PII** o **alucinaciones** en tiempo real, antes de que el output sea consumido por el usuario o por otros microservicios."
