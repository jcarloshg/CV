# SQL Window Functions

## Overview

Window functions perform calculations across a set of rows related to the current row, without collapsing the result set like aggregate functions do. They operate on a "window" of rows defined by the `OVER()` clause.

### Syntax

```sql
window_function_name(expression) OVER (
    PARTITION BY expr1, expr2, ...
    ORDER BY sort_expression [ASC | DESC]
    {ROWS | RANGE} frame_clause
)
```

### Key Components

| Component      | Description                                          |
| -------------- | ---------------------------------------------------- |
| `PARTITION BY` | Divides rows into partitions (optional)              |
| `ORDER BY`     | Defines row order within each partition              |
| `FRAME`        | Defines the subset of current partition (ROWS/RANGE) |

## Most Used Window Functions

### 1. Ranking Functions

#### ROW_NUMBER()

Assigns a unique sequential integer to each row within a partition.

```sql
SELECT
    employee_id,
    department_id,
    salary,
    ROW_NUMBER() OVER (PARTITION BY department_id ORDER BY salary DESC) as row_num
FROM employees;
```

#### RANK()

Assigns a rank with gaps (same rank for ties, next rank skips).

```sql
SELECT
    employee_id,
    salary,
    RANK() OVER (ORDER BY salary DESC) as rank
FROM employees;
-- Output: 1, 1, 3, 4 (ties share rank, next skips)
```

#### DENSE_RANK()

Assigns ranks without gaps (same rank for ties, next rank is sequential).

```sql
SELECT
    employee_id,
    salary,
    DENSE_RANK() OVER (ORDER BY salary DESC) as dense_rank
FROM employees;
-- Output: 1, 1, 2, 3 (ties share rank, next is sequential)
```

#### NTILE(n)

Distributes rows into `n` roughly equal groups (1 to n).

```sql
SELECT
    employee_id,
    salary,
    NTILE(4) OVER (ORDER BY salary DESC) as quartile
FROM employees;

-- Output (with 8 employees, salaries: 100k, 90k, 80k, 70k, 60k, 50k, 40k, 30k):
-- employee_id | salary  | quartile
-- ------------+---------+----------
-- 1           | 100000  | 1
-- 2           | 90000   | 1
-- 3           | 80000   | 2
-- 4           | 70000   | 2
-- 5           | 60000   | 3
-- 6           | 50000   | 3
-- 7           | 40000   | 4
-- 8           | 30000   | 4
```

### 2. Value Functions

#### LAG(column, offset, default)

Returns value from the `n`th row before the current row.

```sql
SELECT
    month,
    sales,
    LAG(sales, 1, 0) OVER (ORDER BY month) as prev_month_sales
FROM monthly_sales;

-- Output:
-- month | sales  | prev_month_sales
-- ------+--------+------------------
-- Jan   | 10000  | 0          (default for first row)
-- Feb   | 15000  | 10000
-- Mar   | 12000  | 15000
-- Apr   | 18000  | 12000
-- May   | 20000  | 18000
```

#### LEAD(column, offset, default)

Returns value from the `n`th row after the current row.

```sql
SELECT
    month,
    sales,
    LEAD(sales, 1, 0) OVER (ORDER BY month) as next_month_sales
FROM monthly_sales;

-- Output:
-- month | sales  | next_month_sales
-- ------+--------+-------------------
-- Jan   | 10000  | 15000
-- Feb   | 15000  | 12000
-- Mar   | 12000  | 18000
-- Apr   | 18000  | 20000
-- May   | 20000  | 0          (default for last row)
```

#### FIRST_VALUE(column)

Returns the first value in the partition.

```sql
SELECT
    employee_id,
    department_id,
    salary,
    FIRST_VALUE(salary) OVER (PARTITION BY department_id ORDER BY salary) as lowest_salary
FROM employees;

-- Output (partitioned by department, ordered by salary ASC):
-- employee_id | department_id | salary  | lowest_salary
-- ------------+---------------+---------+---------------
-- 103         | 10            | 3000    | 3000
-- 101         | 10            | 5000    | 3000
-- 102         | 10            | 7000    | 3000
-- 201         | 20            | 4000    | 4000
-- 202         | 20            | 6000    | 4000
-- 203         | 20            | 8000    | 4000
```

#### LAST_VALUE(column)

Returns the last value in the partition (requires careful frame specification).

```sql
SELECT
    employee_id,
    department_id,
    salary,
    LAST_VALUE(salary) OVER (
        PARTITION BY department_id
        ORDER BY salary
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) as highest_salary
FROM employees;

-- Output (partitioned by department, ordered by salary ASC):
-- employee_id | department_id | salary  | highest_salary
-- ------------+---------------+---------+---------------
-- 103         | 10            | 3000    | 7000
-- 101         | 10            | 5000    | 7000
-- 102         | 10            | 7000    | 7000
-- 201         | 20            | 4000    | 8000
-- 202         | 20            | 6000    | 8000
-- 203         | 20            | 8000    | 8000

-- Note: Without frame spec, default is RANGE UNBOUNDED PRECEDING,
-- which would return the current row's value instead of partition's last!
```

### 3. Aggregate Functions as Window Functions

These work like regular aggregates but preserve individual rows:

```sql
SELECT
    employee_id,
    department_id,
    salary,
    SUM(salary) OVER (PARTITION BY department_id) as dept_total,
    AVG(salary) OVER (PARTITION BY department_id) as dept_avg,
    COUNT(*) OVER (PARTITION BY department_id) as dept_count,
    MAX(salary) OVER (PARTITION BY department_id) as dept_max,
    MIN(salary) OVER (PARTITION BY department_id) as dept_min
FROM employees;
```

#### Running Total Example

```sql
SELECT
    date,
    amount,
    SUM(amount) OVER (ORDER BY date) as running_total
FROM transactions;
```

## Common Frame Specifications

| Frame                                                      | Description                            |
| ---------------------------------------------------------- | -------------------------------------- |
| `ROWS UNBOUNDED PRECEDING`                                 | From start of partition to current row |
| `ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING` | Entire partition                       |
| `ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING`                 | Current row + 1 before + 1 after       |
| `ROWS CURRENT ROW`                                         | Only current row                       |

## Practical Use Cases

### Calculate Year-over-Year Growth

```sql
SELECT
    year,
    revenue,
    LAG(revenue, 1) OVER (ORDER BY year) as prev_year_rev,
    revenue - LAG(revenue, 1) OVER (ORDER BY year) as yoy_growth
FROM annual_revenue;

-- Output:
-- year | revenue | prev_year_rev | yoy_growth
-- -----+---------+---------------+------------
-- 2020 | 100000  | NULL          | NULL
-- 2021 | 120000  | 100000        | 20000
-- 2022 | 115000  | 120000        | -5000
-- 2023 | 140000  | 115000        | 25000
-- 2024 | 160000  | 140000        | 20000
```

### Find Top N Per Group

```sql
SELECT * FROM (
    SELECT
        department_id,
        employee_id,
        salary,
        ROW_NUMBER() OVER (PARTITION BY department_id ORDER BY salary DESC) as rn
    FROM employees
) WHERE rn <= 3;

-- Output (Top 3 highest-paid employees per department):
-- department_id | employee_id | salary  | rn
-- --------------+------------+---------+----
-- 10            | 102        | 9000    | 1
-- 10            | 101        | 7000    | 2
-- 10            | 103        | 5000    | 3
-- 20            | 205        | 8500    | 1
-- 20            | 201        | 6000    | 2
-- 20            | 202        | 5500    | 3
```

### Cumulative Distribution

```sql
SELECT
    employee_id,
    salary,
    CUME_DIST() OVER (ORDER BY salary) as cumulative_pct
FROM employees;
```

## Summary Table

| Function                          | Type      | Purpose                  |
| --------------------------------- | --------- | ------------------------ |
| `ROW_NUMBER()`                    | Ranking   | Unique sequential number |
| `RANK()`                          | Ranking   | Rank with gaps           |
| `DENSE_RANK()`                    | Ranking   | Rank without gaps        |
| `NTILE(n)`                        | Ranking   | Divide into n groups     |
| `LAG()`                           | Value     | Previous row value       |
| `LEAD()`                          | Value     | Next row value           |
| `FIRST_VALUE()`                   | Value     | First value in partition |
| `LAST_VALUE()`                    | Value     | Last value in partition  |
| `SUM()/AVG()/COUNT()/MAX()/MIN()` | Aggregate | Aggregate calculations   |

---

## Key Differences

- **Aggregate vs Window**: Aggregate collapses rows; Window preserves them
- **RANK vs DENSE_RANK**: RANK has gaps; DENSE_RANK doesn't
- **LAG vs LEAD**: LAG looks back; LEAD looks forward
- **PARTITION BY**: Without it, entire result set is one partition
